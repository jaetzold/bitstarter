The core interface will just be REST with JSON. Playing it this way would be a bit nerdy, sure. But we want to focus on the core logic, not the presentation.

Oh, and it will be build with some kind of functional language, of course.

At the same time this should be ideal to allow to build specialized clients presenting the information in an appealing way to a player. You're welcome to contribute!

FAQ

What does MUD stand for?
Multi User Dungeon. Not necessarily but commonly this means a text based role-playing adventure on the internet where multiple users can play with different characters in the same game world at the same time.
Why REST based and not telnet as many others?
Because we're nerds and just like it to do things a bit differently. We also think that we'll get some more generally useful insights along the way.
